# Contributing to Music Data Platform

We welcome contributions! Please follow these steps:

1.  **Fork the repository**.
2.  **Create a feature branch**: `git checkout -b feature/my-new-feature`.
3.  **Commit your changes**: `git commit -m 'Add some feature'`.
4.  **Push to the branch**: `git push origin feature/my-new-feature`.
5.  **Submit a pull request**.

## Standards

*   **Code Style**: We use `black` for Python formatting.
*   **Tests**: Ensure all new features are covered by tests. Run `make test` before submitting.
*   **Infrastructure**: All infrastructure changes must be reflected in Terraform code.
*   **CI**: Keep `.github/workflows/ci.yml` green for pull requests.

## Reporting Issues
Please use the GitHub Issues tab to report bugs or request features.
